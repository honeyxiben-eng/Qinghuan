# 智慧盐湖数据管理平台 · 功能开发文档

> 版本：v3.0
> 日期：2026-06-09
> 技术栈：Next.js 16 (Turbopack) + TypeScript + SQLite（本地 Prisma） + TailwindCSS

本文档仅描述业务功能，不涉及 UI 实现细节。

---

## 一、系统概述

智慧盐湖数据管理平台是一套面向盐湖采卤井的全生命周期数据管理系统。系统围绕**井基础信息、动态监测数据、卤水化验数据**三大数据域，提供数据录入、浏览查询、筛选导出、数据分析与异常预警等功能。

### 1.1 用户角色

| 角色 | 权限标识 | 说明 |
|------|---------|------|
| 管理员 | admin | 拥有全部权限，可访问所有页面与功能 |
| 化验员 | lab | 可录入与管理化验数据 |
| 采卤员 | brine | 可录入与管理监测数据 |

默认账户：admin / admin123（管理员）、lab01 / lab123（化验员）、brine01 / brine123（采卤员）

### 1.2 系统初始化

首次启动时自动创建 users 表并插入三个默认账户。JWT 用于后续 API 鉴权（12 小时有效期）。

### 1.3 运行时环境

| 规范项 | 值 |
|------|------|
| 运行时 | Node.js |
| 框架 | Next.js 16 (Turbopack) |
| 语言 | TypeScript |

## 二、认证与会话

### 2.1 登录
- 用户名 + 密码登录
- 密码使用 bcryptjs 哈希存储，不可逆
- 登录成功后生成 JWT Token，写入 Cookie（sl_role、sl_user），有效期 7 天
- Token 同时在服务端以 JWT 签发，供 API 鉴权使用（12h 有效期）

### 2.2 登出
- 清除客户端 Cookie
- 清除 Zustand 全局状态中的用户信息

### 2.3 全局状态（Zustand）
- userRole / userName：当前登录用户
- refreshSec：数据自动刷新间隔（存 localStorage，默认 30s）
- sidebarCollapsed：侧边栏折叠状态（存 localStorage）
- 页面加载时自动从 Cookie/localStorage 水合（hydrate）状态

### 2.4 API 鉴权
- API 路由通过 JWT 中间件验证身份
- REST API 和 Server Actions 共享同一 Service 层

## 三、基础信息管理（井档案）

### 3.1 数据模型

WellLineInfo（采卤线） 1:N WellInfo（井）

每口井属于一条采卤线。系统预置 22 条采卤线，覆盖北部、中部、西部、东部、南部五个片区。

井字段：井号、所属采卤线、成井日期、成井工艺、工艺备注、井口尺寸、初见水位、设计井深、坐标 X/Y

### 3.2 功能列表

| 功能 | 说明 |
|------|------|
| 井列表查询 | 分页查询，每页 20 条。支持按片区、采卤线、井号关键词筛选 |
| 井详情查看 | 查看单井完整信息，含所属采卤线、片区归属 |
| 新增井 | 录入井号、所属采卤线及全部字段 |
| 编辑井 | 修改成井日期、工艺、井口尺寸、水位、井深、坐标 |
| 批量删除井 | 多选后批量删除 |
| 导出井 Excel | 导出当前筛选结果（无筛选则全部），含井号、采卤线、片区等 11 列 |
| 导入井 Excel | 上传 Excel 批量导入，列名别名容错，井号+采卤线为必填 |

### 3.3 导入容错

列名通过别名表匹配。例如：井采线也匹配采线；成井时间也匹配成井日期。
